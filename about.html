About Page Replacement Algorithms

Page replacement algorithms are essential in operating systems to decide which memory pages to swap out when new pages need to be loaded into memory. These algorithms aim to minimize page faults and optimize system performance.

Our simulator helps visualize three fundamental page replacement algorithms: FIFO (First-In-First-Out), LRU (Least Recently Used), and ESC (Enhanced Second Chance). Below you'll find detailed explanations of each algorithm.

FIFO (First-In-First-Out)

The FIFO algorithm replaces the oldest page in memory, following a simple queue-based approach. When a new page needs to be loaded and all frames are occupied, the page that has been in memory the longest is replaced.

How it works:

  1. New pages are added to the tail of the queue
  2. When a page fault occurs, the page at the head is replaced
  3. The newly loaded page is added to the tail

Characteristics:

  • Simple to implement (uses a queue)
  • Low overhead
  • May suffer from Belady's anomaly (more frames can lead to more page faults)
  • Doesn't consider page usage frequency or recency

Example Scenario

1
Page 1 loaded
2
Page 2 loaded
3
Page 3 loaded
4
Page 4 replaces Page 1
(Assuming 3 frames and reference string: 1, 2, 3, 4, 1)

LRU (Least Recently Used)

The LRU algorithm replaces the page that hasn't been used for the longest period of time. It's based on the principle of temporal locality - pages used recently are likely to be used again soon.

How it works:

  1. Each page access updates its "last used" timestamp
  2. When a page fault occurs, the page with the oldest timestamp is replaced
  3. New pages or accessed pages move to the "most recently used" position

Characteristics:

  • More complex to implement than FIFO
  • Generally better performance than FIFO
  • Doesn't suffer from Belady's anomaly
  • Requires maintaining usage history (can be hardware-assisted)

Example Scenario

1
Page 1 loaded
2
Page 2 loaded
3
Page 3 loaded
1
Page 1 accessed again
4
Page 4 replaces Page 2
(Assuming 3 frames and reference string: 1, 2, 3, 1, 4)

ESC (Enhanced Second Chance)

The Enhanced Second Chance algorithm is a more sophisticated version of the Clock algorithm that considers both reference and modify bits to make replacement decisions. It's commonly used in practical systems.

How it works:

  1. Each page has a reference bit (R) and modify bit (M)
  2. Pages are classified into four categories: (0,0), (0,1), (1,0), (1,1)
  3. The algorithm scans pages in a circular fashion (like a clock)
  4. It looks for a page with (0,0) first, then (0,1), etc.
  5. During scan, R bits are cleared to identify pages not recently used

Characteristics:

  • Approximates LRU with less overhead
  • Considers both usage and modification status
  • Good balance between performance and implementation complexity
  • Commonly used in real operating systems

Example Scenario

1
Page 1 loaded (R=1, M=0)
2
Page 2 loaded (R=1, M=1)
3
Page 3 loaded (R=0, M=0)
4
Page 4 replaces Page 3
(Assuming 3 frames and Page 3 is (0,0))
Bit Meanings:
(0,0): Not used, not modified
(0,1): Not used, modified
(1,0): Used, not modified
(1,1): Used and modified
